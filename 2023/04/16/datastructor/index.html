有空間限制的裝置取第k個大小的數值

  • 如果有空間限制就不能全部讀進陣列進行排序
  • 所以取k個大小的陣列就好
  • 裡面用排序需要nlog(k)、直接調整位置只要2k
#include 
using namespace std;

int main() {
    int inpt[] = {-20, 9, 15, 6, 17, 30};
    int size_ = sizeof(inpt)/sizeof(int);

    int n;
    int k = 3;
    int* arr = new int[k]();
    for (int i = 0; i < k; ++i)
        arr[i] = INT32_MIN;

    for (int i = 0; i < size_; ++i) {
        n = inpt[i];
        // 只有n比最小的第k數字大才需要有動作
        if (n > arr[k-1]) {
            // 找到n應該放的位置
            int pos = 0;
            for (int j = k-1; j >= 0; --j) {
                if (n <= arr[j]){
                    pos = j;
                    break;
                }
            }
            // 將比n小的數值都往後移動
            for (int j = k-1; j > pos; --j)
                arr[j] = arr[j-1];
            // 將n放入
            arr[pos] = n;
        }
    }

    for (int i = 0; i < k; ++i)
        cout << arr[i] << " ";
    cout << "\n";
    cout << arr[k-1] << endl;

    system("pause");
    return 0;
}

時間&空間複雜度

空間複雜度

  • 全部空間 = S( P ) = c + Sp(I)
    c: 與輸入輸出無關的變數空間(例如變數宣告。
    Sp(I): 輸入影響到的空間，==如果輸入不會影響到變數空間變化就是0。==

EX:

Type Name number of bytes
int之類的 a 4
指標 list 4

時間複雜度

  • T( P )= c + Tp(I)
    • c: 編譯時間
    • Tp(I): 根據I覺得地運行時間
  • 表列法
    • 確定每個指令需要的步驟數 步驟/指令(steps/execution or s/e)
    • 每個指令被執行的次數(==頻率frequency==)。
    • s/e乘上頻率f就是每個指令被執行次數。

EX:

Statement s/e frequency Totle steps
for(int i=0;i<n;++i); 1 n+1 n+1
int i = 0; 1 1 1

  • Big O define

    • f(n) <= c * g(n)。
    • ex: f(n)=3*n + 2, g(n)=n, c=4 成立。
    • g(n)必須是最小上界。

  • 漸進符號-Omega define

    • f(n)=Ω(g(n))
    • f(n) >= c * g(n)
    • ex: f(n)=3n+3, g(n)=n, c=3
    • 3n+3 >= 3n
    • g(n)必須是最大下界

  • 漸進符號-Theta define

    • c1g(n) <= f(n) <= c2g(n)
    • g(n)必須符合最小上界和最大下界。

矩陣

稀疏矩陣(sparse matrix)

  • 使用三元序對進行儲存。
  • 對於一個三元序對，行為i，列為j，值為v。需將其i與j的值對調才能得到新的三元表，但是如果直接進行轉換，得到的新的三元序對的順序是混亂的，不符合三元序對的規則。
  • 先計算所有舊的col位置的數量，再用prefix sum計算他們位置。
  • 走訪所有元素根據prefix sum位置放置到新的矩陣位置。
